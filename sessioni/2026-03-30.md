## Blocco attuale: permessi TCC Reminders su macOS 26

### Problema
Su macOS 26, quando Node.js chiama `execFile('osascript', ...)` o il binario Swift come child_process, il processo viene terminato silenziosamente (exit code null, nessun signal catturato, stdout/stderr vuoti). Il dialog TCC per "Promemoria" non appare mai.

### Cosa funziona
- `osascript` eseguito come background task da Claude Code → ha ottenuto il permesso (dialog approvato dall'utente) → restituisce dati corretti
- Il client React carica le **liste** dal dropdown (getLists funziona) ✅
- L'app è accessibile via Tailscale: `http://100.102.75.19:5173` ✅
- Privacy > Promemoria mostra: Calendr.app, sshd-keygen-wrapper (non osascript, non Terminal)

### Tentavi falliti
1. Firma Swift CLI con `codesign --sign - --entitlements reminders.entitlements`
2. Bundle identifier esplicito `com.avvocati-e-mac.reminders-kit`
3. App bundle completo (RemindersKit.app) con Info.plist + NSRemindersUsageDescription
4. Sostituzione Swift CLI con JXA via osascript + file /tmp/
5. Far avviare il server dal Terminale fisico dell'utente

### Stato file chiave
- `server/src/services/remindersService.js` → usa osascript/JXA con file /tmp/
- `server/RemindersKit.app/` → bundle creato ma non usato
- `server/bin/reminders-kit` → binario Swift firmato (non usato)
- `cli/reminders.entitlements` → entitlements file
